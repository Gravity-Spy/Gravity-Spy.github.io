Running wscan on the command line¶

As described on the home page, the main interface to the gravityspy algorithm is the command-line executable wscan.

Basic instructions¶

Basic instructions on running wscan can be gained by looking at the --help menu:

$ wscan --help
/Users/scottcoughlin/Documents/virtualenvs/GravitySpy-py35/lib/python3.5/site-packages/h5py/__init__.py:36: FutureWarning: Conversion of the second argument of issubdtype from `float` to `np.floating` is deprecated. In future, it will be treated as `np.float64 == np.dtype(float).type`.
  from ._conv import register_converters as _register_converters
Using Theano backend.
usage: wscan [-h] [-V] --inifile INIFILE --eventTime EVENTTIME [--uniqueID]
             [--ID ID] [--outDir OUTDIR] --pathToModel PATHTOMODEL [--runML]
             [--verbose] [--HDF5] [--PostgreSQL]

optional arguments:
  -h, --help            show this help message and exit
  -V, --version         show program's version number and exit
  --inifile INIFILE     Name of ini file of params
  --eventTime EVENTTIME
                        Trigger time of the glitch
  --uniqueID            Is this image being generated for the GravitySpy
                        project, if so you must assign a uniqueID string to
                        label the images instead of GPS time
  --ID ID               The uniqueID string to be supplied with --uniqueID
  --outDir OUTDIR       Outdir for images
  --pathToModel PATHTOMODEL
                        Path to folder containing trained model
  --runML               Run the ML classifer on the omega scans
  --verbose             Run in Verbose Mode
  --HDF5                Store triggers in local HDF5 table format
  --PostgreSQL          Store triggers in a remote PostgreSQL DB
